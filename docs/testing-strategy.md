# Testing Strategy

This document will track the testing approach for PitWall.

For the current full planning reference, see:

- [`project-plan.md`](./project-plan.md)

## Testing Layers

- Unit testing
- Mock live feed testing
- Backend integration testing
- Frontend component testing
- End-to-end testing
- Deployment testing
- Security testing
- Manual race-day testing

## Key Decision

Mock mode should be built early so the dashboard can be tested without a live F1 session.
